# URR — 프로젝트 개요 & 비즈니스 규칙

## 서비스 정의

K-POP 찐팬을 위한 공정 티켓팅 플랫폼. 티켓팅 → 양도 → 커뮤니티 통합.
매크로·봇 차단 + 멤버십 등급으로 팬 활동을 정량화해 티켓 우선권 부여.

**가격**: 연간 ₩30,000 / 아티스트별 가입

---

## 멤버십 등급 & 규칙

| 등급            | Lv. | 조건             | 예매 창       | 예매 수수료 | 양도 수수료 |
| --------------- | --- | ---------------- | ------------- | ----------- | ----------- |
| 🌩️ 라이트닝     | 4   | FTS ≥ 85점       | 1순위         | 없음        | 5%          |
| ⚡ 썬더          | 3   | FTS 66–84점      | 2순위 (+1h)   | +₩3,000     | 5%          |
| ☁️ 클라우드     | 2   | 멤버십 가입 즉시 | 3순위 (+1day) | +₩5,000     | 10%         |
| 🌫️ 미스트       | 1   | 회원가입 즉시    | 일반예매만    | +₩8,000     | 10%         |

**핵심 규칙**

- 라이트닝/썬더: 대기열 직행 (Fast Track)
- 클라우드/미스트: 대기열 진입 (Standard Path)
- 양도 판매/구매: 해당 아티스트 **멤버십 보유자(클라우드 이상)만** 가능
- 멜론 미연동 시 미스트 유지. 연동 시에만 상위 등급 가능
- 멜론 연동은 강제 게이트 아님 — "상위 등급 잠금 해제" CTA 형식

## MVP 범위

**IN**: 홈, 검색, 아티스트 페이지, 공연 목록/상세, 예매 플로우(대기열+좌석+결제), 멤버십 가입, 마이페이지(티켓 월렛+멤버십+양도내역), 양도 마켓, 온보딩, 알림(UI 쉘)

**OUT**: 커뮤니티 피드, 실시간 채팅, 푸시 알림 실제 트리거, 모바일 반응형, 멜론 실제 API 연동(Mock으로 대체)

---

## 온보딩 플로우 (8단계)

1. 소셜 로그인 (Kakao / Naver) + 이메일
2. **본인인증** (CI 1인1계정): 통신사 선택 → 이름/생년월일/성별/전화번호 → SMS 코드(3분 타이머) → CI 중복 시 차단
3. **약관 동의**: [필수] 이용약관 / [필수] 개인정보 / [선택] 마케팅
4. 가입 완료 → **미스트 즉시 부여**
5. 아티스트 선택 (1명 이상 필수)
6. 멤버십 소개: 등급 혜택 비교표 + [가입 ₩30,000/년] or [나중에]
7. 멜론 연동 (선택): "상위 등급 확인" CTA
8. 완료 → 개인화된 홈

## 양도 마켓 규칙

- 가격 범위: 정가 **0.5x ~ 1.5x** (벗어나면 에러)
- 에스크로: 구매자 결제 → 플랫폼 보관 → 소유권 이전 → 판매자 지급
- `listed` 상태만 수정/취소 가능
- 신뢰 지표: 판매자 티어 배지 + 거래 건수 표시

## 예매 상태머신

```
idle
  └─[Book Now]─→ queue → seats-section → seats-individual → payment → confirmation
                                        ↘ seats-expired(3분 초과) → seats-section 복귀
                                                               payment-failed → 60s 재시도
```

**좌석**: 최대 4석, 3분 타이머 (1:00→노랑, 0:30→빨강+pulse)
**대기열**: 순번+예상대기+성공확률%, 10초 WebSocket 갱신
