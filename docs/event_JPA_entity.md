# URR 이벤트 서비스 — JPA Entity & 시드 데이터 SQL

---

## SQL 재생성 프롬프트

docs/event_JPA_entity.md에 정의된 테이블 구조를 기준으로,
docs/event.sql 파일을 업데이트 해주세요.

[데이터 생성 우선순위]

1. 제공된 API 응답 예시 데이터 (있는 경우)
2. src/shared/lib/mocks/ 디렉토리의 데이터
3. 위 데이터가 부족한 경우, 테이블 구조를 기반으로 샘플 데이터 생성

[전제 조건]

- 테이블은 Hibernate를 통해 이미 생성된 상태
- MySQL 기준 SQL 문법
- Entity 필드명 = DB 컬럼명

[규칙]

- null 값은 명시적으로 NULL로 작성
- enum은 문자열 그대로 저장
- 날짜/시간은 MySQL DATETIME 형식
- 외래키는 실제 데이터 관계 기준으로 매핑 (mock 또는 API 기준)
- bulk insert 형태로 작성

[공통 컬럼]

- created_at, updated_at 포함하여 작성

[출력]

- docs/event.sql 파일 전체를 교체
- 테이블 단위로 구분 (-- 주석)
- 실행 가능한 SQL만 작성
